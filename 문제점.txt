문제점

1. 임베드 게시로 게시된 임베드에는 웹사이트 버튼이 없어야 한다.

2. 디스코드.js 컴포넨트 V2가 적용되지 않았다.

3. 입금 신청 모달이 잘 작동하지 않는다.(예를 들어, 숫자 입력란에 글을 쓰고 모달을 제출하면 금액을 적어주세요 라는 오류 메시지가 아닌 무언가 잘못되었다는 메시지만 떠 헷갈리므로 로그를 명확히 보여줘야 한다.)

4. 후기 버튼을 누르면 상호작용 실패가 뜬다
   카테고리 추가 후 상품 버튼을 누르면 상호작용 실패

5. 구매 등등 기능 잘 작동하는지 확인

6. 고급스러운 딥 블랙(Deep Black)과 그에 대비되는 화이트를 사용해서 웹사이트를 리디자인해줘

그리고 배경은 검은색 계열로 하고, 텍스트는 눈이 아프지 않은 밝은 회색이나 흰색으로 대비를 맞춰서 CSS 코드 짜줘.

7. 임베드 테스트 기능 안됨(버튼)

8. 임베드 설정에서 사진이 적용되지 않음. (사진 하나만 넣을 수 있도록 수정)

9. 자충 안됨

10. 자충 성공시 성공되었다는 메시지가 없음, 시간 제한이 없음

11. 수동으로 충전 문의 수락할 때 버튼의 박스 색이 흰색임(배경이랑 맞지 않음)

12. interactionCreate.js에서 가능별로 좀 구분해서 파일 나누는것도 좋을듯.

14. 구매할 때 임베드에 상품 버튼 누르면 그 이후 모든 메시지는다 에피메럴 처리로 수정하기. +후기 또한 마찬가지.

15. 카테고리 삭제 제품 삭제 등등 삭제/수정 로직 추가 

16. 내 대시보드 버튼 위치 수정하고, id나오는거 대신 자기 프로필 사진이랑 사용자명이 나오도록 수정하기, 로그아웃 버튼이랑 내 대시보드 버튼이랑 css스타일은 동일하게 설정하기.

17. .env파일에 있는 서버id와 링크의 서버에 들어와있지 않은 사람들이 사이트에 로그인하면 안내 문구로 서버링크를 보내주고, 서버에 먼저 참여해주세요 라는 문구를 보여준다.

18. 웹사이트에서의 모든 인풋란은 브라우저의 프롬프트 기능이 아니라, 모달을 띄워서 설정하도록 수정하기.

19. 자충 음수 가능하게 수정하기 0원도 가능하도록

20. 구매로그, 수동충전 등의 로그가 안뜸.(웹사이트, 채널 둘 다)

21. 재고 입고 방식을 재고 수정을 눌렀을때 줄바꿈으로 새 재고를 입력할 수 잇는 모달창을 생성하는것으로 수정

22. 공용 웹사이트용 모달양식을 만들어서 기존 원도우 프롬프트를 이용해 답변을 받는 것을 모달을 이용할 수 있도록 수정하기.


명령 프롬프트 

윈도우 운영체제이고 한국어를 사용 중이며 파워쉘 환경에서 실행 중이다.



문제점.txt를 읽고 각 번호별로 순차적으로 진행하되, 한 번호가 끝나면 나에게 허락을 맡고 기능을 테스트한다.
